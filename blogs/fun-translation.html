Behind The Scenes: Fun Translation Web App

This is a fun API based language translation app. Use this to translate english language to some other funny ones and have fun.

Back Story:

So, as I have done all the lectures of Neog Camp, now it's time to build all the projects and submit all of them for review. I have already completed most of them the only lefts are react one and then fun translation one. So, today I will be building the fun translation web app. So, talking about it's functionality it will basically work on the API calls it will send a text to this website using it's APIs and then receive a response in JSON format and then retrive the required data and show it to the user. So, let's see what are the requirements for this web app.

Requirements:

So, first let's browse through the website and get some APIs. So, I have decided to use the 'pirate, yoda, navi and groot' API. The second thing is to have a basic design stucture in the mind so that it's easier to code. And I don't think we need anything else as rest is coding part the most enjoying and interesting thing so let's begin with the code.

Main Process:

So, I will be using the simple approch as used in all the previous ones which is to break the problem into smaller chunks and then work onto them. Starting by initializing a repository, I have created a file index.html. Now I will tell you the idea and stucture of the web app so I will be using 4 languages ie., 4 APIs so the user will first select the language, then enter the text to convert and click on the translate button and in the backend an API call will be made based on the language selected and the converted text will be displayed to the user. So, now starting with the nav bar first, very simple using nav tag and h1 tag. By the way I will not work on the styling part now, but build the structure first. Now creating a dropdown menu to select the language using select tag next I have created textarea, a button, again a textarea and a simple div container for the about section. Now time for the footer, so this is also really simple using footer tag now I have commit the changes. Now time for the styling part starting with creating a directory named css and inside that a style.css file and link it to the index.html file. So, firstly importing the fonts from Google Fonts and then defining the body tag margin and padding both 0 and now defining two color variables for main color and another for the hover state now giving the nav bar some padding, border, color and align the text to centre. So, the process is same for all the elements so I will not document it and I will directly start the JavaScript file to give the web app functionality and commit all the changes. So, again starting with creating a directory js and a file myapp.js inside it. So, starting with getting the elements first, we can use either method queryselector or getelementbyid for that, here I will use getelementbyid and grab the dropdown menu both the textarea and the button now with the main logic and fuctions so, I have defined a fuction to set the API based on the language selection and return the final API. Now for the second fuction it will grab the value of the textarea and assign it to a variable and then return the final API with the argument parsed. Now defining a fuction for error handling which will console log the the error message and also show an alert to the user. Now comes the main fuction which later be used as the on click event listner fuction to the button. Here first I will call the setAPI fuction and pass it's return value to the serveAPI fuction and now we will use the fetch fuction to fetch the response of the API by sending a http request and then chaining a then and parsing the response as JSON and again chaining a then and assign the converted text to the the textarea by using the innertext and now finally assigning an event listner to the button which will be triggered when the button is clicked and will call our fuction. And hence this completes my project now committing all the changes and pushing it to remote repository on GitHub. Now I will host the web app using Netlify so after signing into it I have selected the new site option from GitHub and selecting the respective repository and now going into the site settings and changing it's name to fun-translations.netlify.app.

So, this was my project for the markSeven submission for Neog Camp.
