Behind The Scenes: Auto Blog Post

A simple Python Automation program to automate the process of writing the blog post HTML code

Back Story:

So, I decided to do live blogging while creating any project with the motive of guiding you through the process of creating a project, thinking process, problems faced and etc. And upload these blogs on my portfolio website. So, to upload it I need to write the HTML code for each blog post and that too has the same code except for the blog content. So, to avoid this repetion and to save time I am making this Python program that will do all of my things automatically and save my lots of time.

Requirements:

So, this app must take a txt file with the raw blog written in it and read its contents and append it to the predefined HTML code. And for the headings and description I am thinking of using the regular expression to match certain keyword in the blog file and put it at their approptiate place. So, I am thinking of using '%h%, %d%' something like this and then match it using the 'Re' Module available in Python.

Main Process:

So, starting with importing the modules I have defined a function to read and store the raw blog content from the file to a variable. Now for writing the the complete code to a file I need to get the file name so I need to figure out whether that will be given by the user or it should be given in the raw blog itself. So, I decided to give it in the raw blog file so that it remains automatic. Now what I need to do is to figure out a way to place the title, header and description to their respective place in the pre-defined HTML code. So, for the ease I have defined another function to process the code and place the required things and I have moved the defination of the header and footer code inside the same function so that when I get the things using regular expression I will use f string to place all of them at their respective place. Now I need to define a fuction to write all the data to a HTML file. So, I tested this application against a sample file and it is working so far and now I'm gonna test this against this blog to check whether it will work against huge file or not. So, the test failed the error is with the regular expression the program ran out of index as the blog is really huge so I need to figure out another way of doing the same thing. I got an idea for doing this so I will be using the built-in string function 'split'. And split it at '%p %' (without any space, space added as the program was also spliting the text here) which is the defination for the main content in the raw blog and I will use the index no 1 as it is the required thing and then rest is the same and now I will test the app again with this same blog. This time the test is successfull. Now I will push the code. The next required thing to do is to move the file form the current directory to the blogs directory the problem here is I use WSL and also Windows for coding and the directory path is different for both the Operating Systems so, now I need to add a condition to check the Operating System and set the path automatically according to the OS this app is running on. So, for that I will be using os.name and set the condition. I can also automate commiting and pushing the code to the portfolio but I won't do that because I can make changes if required.

This blog was added using the Python Auto Blog Post application and it is working correctly.
